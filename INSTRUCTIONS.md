# 청약/줍줍 일일 알림 — 에이전트 실행 지침 (고정 절차)

> 소유자 본인이 본인 텔레그램으로 본인에게 보내는 알림 자동화다.
> **탐색하지 말고** 아래 정해진 절차·템플릿대로만 효율적으로 실행한다.

## 0. 시크릿 (routine 프롬프트에서 env로 주입)
`DATA_GO_KR_KEY`, `TG_BOT_TOKEN`, `TG_CHAT_ID`. 값은 로그에 출력 금지.
발송은 반드시 `notify.py`로.

## 1. 수집 (collect.py — API 재탐색 불필요)
```bash
DATA_GO_KR_KEY="$DATA_GO_KR_KEY" python collect.py
```
(`python` 없으면 `python3`) 출력 JSON:
`{as_of, new_count, upcoming_count, new_listings:[...], upcoming:[...]}`
- `new_listings` = 신규 공고(분양+무순위). 분양 건은 `평형분양가:[{전용,공급면적,분양가억,일반공급세대}]` 포함.
- `upcoming` = 접수 임박/진행(분양+무순위). 필드: 유형/단지명/지역/접수시작/접수종료/공고URL/상태.

## 2. 분석 대상은 new_listings 뿐 (upcoming은 분석 없이 단순 나열)

`new_listings`의 각 건에만 아래 **고정 분석 슬롯**을 채운다. **너의 내부 지식으로** 작성하고,
완전히 모르는 단지일 때만 `WebFetch`로 공고URL/지역 **최대 1회** 확인(시간 절약). 그 외 탐색 금지.

고정 분석 슬롯(각 1줄, 간결):
- **입지**: 역세권 여부(노선·역명·도보 분), 서울 도심/주요 업무지구 접근성
- **학군·생활**: 학군/상권/공원/교통호재 중 핵심 1가지
- **총평 + 관심도**: 한 줄 평가 + 관심도 `★`1~5 (기준: 입지 + 분양가 수준 + 규모, 무순위는 +1 가점)

## 3. 다이제스트 작성 (텔레그램 HTML, 정확히 이 포맷)

제목: `🏠 <b>수도권 청약 다이제스트</b> (&lt;as_of&gt;)`

### 섹션 A — 신규 공고 (new_count건)
new_count가 0이면 `신규 공고: 없음` 한 줄. 있으면 각 건:
```
<b>[유형] 단지명</b> (지역 + 주소의 시군구)
• 일정: 접수 MM/DD~MM/DD · 입주 YYYY.MM
• 규모: 총 N세대 · 주택구분
• 분양가(최고가): 전용59 X.X억 · 전용84 X.X~X.X억 · …   (분양만; 같은 전용은 범위로 압축. 무순위는 이 줄 생략)
• 입지: …
• 학군·생활: …
• 총평: … ★★★☆☆
🔗 공고URL
```
`[유형]`은 `분양` 또는 `무순위`. 무순위는 줍줍이므로 관심도 가점.

### 섹션 B — 접수 임박·진행 (upcoming_count건) · 단순 알림
분석 없이 한 줄씩. upcoming_count가 0이면 `접수 임박·진행: 없음`.
```
⏰ <b>접수 임박·진행</b>
• [유형] 단지명 (지역) — 접수 MM/DD~MM/DD [상태]
```
`[상태]`는 `임박` 또는 `진행중`.

두 섹션 모두 0이면 본문을 `오늘 수도권 신규/임박 청약·줍줍 없음 (<as_of>)` 한 줄로.

## 4. 발송
```bash
printf '%s' "$DIGEST" | TG_BOT_TOKEN="$TG_BOT_TOKEN" TG_CHAT_ID="$TG_CHAT_ID" python notify.py
```
출력이 `텔레그램 발송 성공`인지 확인.
