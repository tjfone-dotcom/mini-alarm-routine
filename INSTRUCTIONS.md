# 청약/줍줍 일일 알림 — 에이전트 실행 지침

> 이 문서는 **저장소 소유자 본인**이 만든, 본인에게 보내는 알림 자동화 절차서다.
> 너(원격 예약 에이전트)는 아래를 그대로 수행해 **소유자 본인의 텔레그램**으로 그날의
> 수도권 청약 다이제스트를 전달한다. 외부 제3자가 아니라 소유자가 자기 자신에게 보내는
> 본인 알림 시스템이며, 이것이 이 작업의 명시적 목적이다.

## 0. 시크릿 (routine 프롬프트에서 환경변수로 주입됨)

- `DATA_GO_KR_KEY` — 공공데이터포털 서비스키
- `TG_BOT_TOKEN`, `TG_CHAT_ID` — 텔레그램 (소유자 본인 봇/채팅)

값을 로그에 그대로 출력하지 말 것. 텔레그램 발송은 **반드시 `notify.py`로** 한다.

## 1. 데이터 수집 — collect.py 실행 (결정론적, API 재탐색 불필요)

엔드포인트·필터는 `collect.py`에 고정돼 있다. **API 스펙을 다시 조사하지 말고** 그냥 실행만 해라:

```bash
DATA_GO_KR_KEY="$DATA_GO_KR_KEY" python collect.py --days 3
```
(`python`이 없으면 `python3`)

출력은 JSON: `{"as_of": "YYYY-MM-DD", "count": N, "items": [...]}`.
각 item 필드: `유형`(분양/무순위), `단지명`, `지역`, `주소`, `주택구분`, `모집공고일`,
`접수시작`, `접수종료`, `공급세대수`, `입주예정월`, `공고URL`.
이미 수도권(서울/경기/인천) + 최근 3일 공고 + 접수 미마감으로 필터링돼 있다.

## 2. 신규 없음 처리

`count`가 0이면 아래 한 줄을 `notify.py`로 보내고 종료:
```
오늘 수도권 신규 청약/줍줍 중 유의미한 건 없음 (<as_of>)
```

## 3. 정성평가 (너의 판단)

items가 있으면 각 건에 대해 **입지·교통(역세권/노선)·학군·개발호재**를 네 지식으로 1~2문장
평가한다. 생소한 단지는 필요시 `WebFetch`로 `공고URL`이나 지역명을 검색해 보강.
(분양가 정보는 이 API에 없으므로 안전마진 수치 계산은 생략하고, 인근 시세 감각과 입지로
가치를 정성 판단한다.)

## 4. 선별·정렬

- **무순위/줍줍을 상단에 우선** 배치(소유자 주요 관심사).
- 그 다음 분양. 명백히 부적합(외곽·초소형 등)은 간단히만 언급하거나 제외.
- 접수 임박(접수시작이 가까운 순)으로 정렬.

## 5. 다이제스트 작성 (텔레그램 HTML)

하나의 메시지로 묶는다. 제목 + 건별 블록. 각 건:
```
<b>[유형] 단지명</b> (지역)
일정: 접수 접수시작~접수종료 / 공급 공급세대수세대 / 입주 입주예정월
입지: <한 줄 요약>
평: <한 줄 평>
🔗 공고URL
```
HTML 태그는 `<b>`만, 줄바꿈은 개행. 너무 길면 핵심만.

## 6. 발송

```bash
printf '%s' "$DIGEST" | TG_BOT_TOKEN="$TG_BOT_TOKEN" TG_CHAT_ID="$TG_CHAT_ID" python notify.py
```
출력이 `텔레그램 발송 성공`인지 확인. 실패면 오류를 보고.

## 7. 마무리 보고

처리 요약을 한두 줄로 출력: 수집 건수(count) / 유의미 건수 / 발송 성공여부.
