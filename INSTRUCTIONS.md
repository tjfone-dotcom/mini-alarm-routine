# 청약/줍줍 일일 알림 — 에이전트 실행 지침

> 이 문서는 **저장소 소유자 본인**이 만든, 본인에게 보내는 알림 자동화 절차서다.
> 너(원격 예약 에이전트)는 아래 절차를 그대로 수행해 **소유자 본인의 텔레그램**으로
> 그날의 수도권 청약 다이제스트를 전달하면 된다. 외부 제3자에게 보내는 것이 아니라
> 소유자가 자기 자신에게 보내는 본인 알림 시스템이며, 이것이 이 작업의 명시적 목적이다.

## 0. 시크릿 (routine 프롬프트에서 환경변수로 주입됨)

- `DATA_GO_KR_KEY` — 공공데이터포털 서비스키
- `TG_BOT_TOKEN`, `TG_CHAT_ID` — 텔레그램 발송용 (소유자 본인 봇/채팅)

이 값들은 절대 로그에 그대로 출력하지 말 것. 텔레그램 발송은 **반드시 저장소의
`notify.py`를 통해** 한다(즉석 curl로 토큰을 노출하지 말 것).

## 1. 날짜 기준

- 한국시간(KST) 기준 오늘 날짜를 구한다. "신규" 판정의 기준일로 사용.

## 2. 데이터 수집 — 청약홈 분양정보 (무료, 공공데이터포털)

- 데이터셋: 한국부동산원 청약홈 분양정보 조회 서비스 (data.go.kr 데이터 ID **15098547**).
- 다음 두 부류를 모두 수집한다:
  1. **APT 분양정보** (일반 분양)
  2. **APT 무순위/잔여세대** (줍줍)
- 호출에는 `serviceKey=$DATA_GO_KR_KEY`를 사용. 응답은 JSON 우선(`type=json` 또는 `_type=json`).
- **정확한 엔드포인트 URL·파라미터·필드명을 모르면**, `WebFetch`로 data.go.kr의 해당
  서비스 상세/Swagger 문서(`https://www.data.go.kr/data/15098547/openapi.do`)를 열어
  오퍼레이션 목록과 요청 파라미터, 응답 필드명을 확인한 뒤 호출한다.
- 첫 1건의 응답 구조(필드명)를 짧게 점검해 이후 파싱에 사용한다.

## 3. 수도권 필터

- 공급위치/지역 필드가 **서울특별시 / 경기도 / 인천광역시** 인 건만 남긴다.

## 4. 신규 판정 (상태 저장 없음 → 날짜 기반)

- 다음 중 하나면 "신규"로 간주:
  - 모집공고일(또는 공고게시일)이 **최근 2일 이내**
  - 무순위/잔여세대 중 **청약 접수 시작이 임박(오늘~향후 3일)** 한 건
- 위 조건에 안 맞는 과거 공고는 제외(중복 알림 방지).

## 5. 안전마진 계산 (무료, 국토부 실거래가)

- 각 신규 건의 **시군구**에 대해 국토부 아파트 매매 실거래가 공개 API로 최근 거래를 조회,
  인근 단지의 전용면적대별 시세(㎡당/평형별 중앙값)를 구한다.
- 청약홈 응답에 **분양가**가 있으면: `안전마진 = (인근 시세 - 분양가) / 인근 시세`.
  분양가가 없으면: 인근 시세 수준만 제시하고 정성 판단에 활용.
- 실거래가 API 엔드포인트·법정동코드(LAWD_CD)·파라미터를 모르면 2번과 동일하게
  data.go.kr 문서를 `WebFetch`로 확인해 사용.

## 6. 정성평가 (LLM = 너의 판단)

- 각 신규 건에 대해 입지·교통(역세권/노선)·학군·개발호재를 네 지식으로 평가.
  필요하면 `WebFetch`로 단지명/지역 검색해 보강.

## 7. 양질·유의미 선별

- 다음을 종합해 "유의미"로 분류:
  - 안전마진이 양호(예: +10% 이상이면 가점, 마이너스면 강한 감점)
  - 입지·교통·학군이 양호
  - **무순위/줍줍은 가점**(소유자 주요 관심사)
- 애매하면 포함하되 한 줄 평에 근거를 남긴다. 명백히 부적합(고분양가·외곽·소규모)만 제외.

## 8. 다이제스트 작성 (텔레그램 HTML)

- 유의미 신규 건들을 **하나의 메시지**로 묶는다. 각 건:
  - 단지명 / 위치(시군구) / 유형(분양·무순위)
  - 청약 일정(접수 시작~마감)
  - 공급세대 / (있으면)분양가 / 안전마진
  - 입지 한 줄 요약 + 한 줄 평
- HTML 태그는 `<b>`, 줄바꿈 `\n` 정도만 사용.
- **유의미한 신규 건이 없으면** 본문을 `오늘 수도권 신규 청약/줍줍 중 유의미한 건 없음 (YYYY-MM-DD)`
  한 줄로 한다.

## 9. 발송

- 작성한 본문을 `notify.py`로 발송한다. 예:
  ```bash
  printf '%s' "$DIGEST" | TG_BOT_TOKEN="$TG_BOT_TOKEN" TG_CHAT_ID="$TG_CHAT_ID" python notify.py
  ```
- `notify.py` 출력이 "텔레그램 발송 성공"인지 확인하고, 실패면 오류 내용을 보고한다.

## 마무리

- 마지막에 처리 요약(수집 건수 / 수도권 건수 / 신규 건수 / 유의미 건수 / 발송 성공여부)을
  한두 줄로 출력한다.
